## 4. Selfie Check

This is the missing part.

### What selfie check does

```txt
Checks:
- Is there a real face?
- Is image quality okay?
- Is it not obviously fake/AI/screenshot?
- Is the same selfie/profile photo used consistently?
```

### What it does NOT do

```txt
It does not decide “woman or not woman.”
It does not auto-reject based on masculine/feminine appearance.
```

Selfie status:

```ts
verification: {
  status: "not_started" | "pending" | "approved" | "needs_review" | "rejected";
  selfieUrl?: string;
  selfieSubmittedAt?: Timestamp;
  selfieCheckProvider?: "manual" | "vendor" | "none";
  selfieCheckResult?: {
    faceDetected: boolean;
    imageQuality: "low" | "medium" | "high";
    suspectedFake: boolean;
    suspectedScreenshot: boolean;
    confidence: number;
  };
}
```

For MVP:

```txt
Option A: Manual-lite
Store selfie → mark pending → admin approves.

Option B: Vendor later
Use Persona / Stripe Identity / HyperVerge / IDfy for stronger checks.
```

Sprint 2 should implement **Option A now**, but keep fields ready for vendor later.

## 6. Trust Engine v1

Trust score is private.

```ts
trust: {
  score: number;
  tier: "new" | "basic_verified" | "trusted" | "host_eligible" | "host";
  verificationStatus: "unverified" | "pending" | "verified" | "restricted";
}
```

Initial scoring:

```txt
Phone verified: +15
Google linked: +5
Username set: +5
Profile complete: +10
Self-declaration accepted: +10
Selfie approved: +25
First event attended: +20
Each attended event: +5
Positive host feedback: +10
Confirmed report: -50
No-show: -10
Fake verification: -100
```
